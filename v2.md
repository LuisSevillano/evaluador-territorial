Convertir la versión actual del proyecto en una **base analítica más fiable y más útil para la exploración territorial**, incorporando:

## 🌦️ 2. Clima real por municipio

### Objetivo

Sustituir placeholders climáticos por datos reales o razonablemente fiables y reproducibles.

### Variables climáticas mínimas recomendadas

Para cada municipio calcular al menos:

- `precip_annual_mm`
- `temp_winter_mean_c`
- `temp_summer_mean_c`

Si es viable, añadir también:

- `temp_jan_mean_c`
- `temp_jul_mean_c`
- `temp_min_winter_mean_c`
- `temp_max_summer_mean_c`

### Enfoque recomendado

Usar el municipio como unidad analítica y calcular el clima a partir de:

#### opción A — v2 simple y suficiente
centroide de cada municipio

#### opción B — v2.5 o posterior
muestreo espacial más fino o agregación sobre polígono

Para esta fase, se recomienda empezar por **centroides**, siempre documentando la limitación.

### Requisitos metodológicos

Cada indicador climático debe documentar:

- fuente
- período temporal
- resolución
- método de agregación
- limitaciones

### Resultado esperado

Añadir al dataset municipal columnas climáticas reales y consistentes.

## 📈 3. Series y estructura para ficha rica

### Objetivo

Preparar no solo métricas resumidas, sino también una pequeña base para gráficos en la ficha municipal.

### Ideal para la ficha

Para cada municipio, disponer de una serie mensual tipo:

- temperatura media mensual
- precipitación mensual

Esto permitiría gráficos como:

- línea de temperatura media por mes
- barras de precipitación mensual
- resumen anual en formato visual

### Formato recomendado

Generar un dataset auxiliar largo, por ejemplo:

```text
municipio_id, month, temp_mean_c, precip_mm
```

o un JSON por municipio si el frontend lo prefiere.

### Entregable sugerido

```text
output/municipios_climate_monthly.csv
```

o equivalente en JSON.

## 🖥️ 6. Frontend — ficha municipal rica

### Objetivo

Enriquecer la ficha municipal para que deje de ser solo descriptiva y pase a ser analítica.

### Contenido mínimo recomendado

#### Bloque 1 — Identificación
- nombre municipio
- provincia
- código
- superficie si está disponible

#### Bloque 2 — Accesibilidad
- bucket de tiempo / isocrona
- indicadores booleanos:
  - dentro de 1h30
  - dentro de 2h00
  - dentro de 2h30
  - etc.

#### Bloque 3 — Clima resumen
- precipitación anual
- temperatura media invierno
- temperatura media verano

#### Bloque 4 — Gráficos
- gráfico de línea: temperatura media mensual
- gráfico de barras: precipitación mensual

#### Bloque 5 — Nota metodológica breve
- fuente del clima
- carácter aproximado si se usa centroide
- fecha o periodo de referencia

### Recomendaciones UX

- no sobrecargar la ficha
- usar un diseño limpio y compacto
- dejar claro qué es dato real y qué es derivado
- mostrar unidades en todas las variables
